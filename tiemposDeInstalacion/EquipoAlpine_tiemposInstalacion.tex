\section{Alpine Linux}
\label{sec:org07e84c6}

\subsection{Montaje del sistema raíz}
\label{sec:org48f8db4}

El mecanismo de instalación de Alpine Linux está constituido por dos scripts alternativa:

\begin{itemize}
\item setup-alpine, el cual puede ser aplicado sobre un disco duro entero y realizará la instalación automáticamente, según las especificaciones dadas por el usuario a través de variables de entorno y preguntas interactivas del mismo script.

\item setup-disk, el cual es aplicado sobre un arbol de archivos ya montado en \textbf{/mnt}. Sobre este sistema se detectan las particiones utilizadas y se realiza la instalación sobre estas mismas.
\end{itemize}

Para nuestro caso de uso, el segundo script será el usado. Por lo que lo primero q que debemos de hacer es crear el arbol raíz en /mnt.

Montamos nuestra partición raíz sobre /mnt con:

\begin{verbatim}
mount -t ext4 <particion> /mnt
\end{verbatim}

Luego, dentro de este, creamos la carpeta /boot con:

\begin{verbatim}
mkdir -p /mnt/boot/efi
mount -t vfat <particion> /mnt/boot/efi
\end{verbatim}

Y finalmente, cargamos la partición swap con:

\begin{verbatim}
swapon <particion>
\end{verbatim}

Una vez hecho esto, tenemos preparado el arbol de archivos sobre el que se instalará el sistema operativo.
\subsection{Instalación de Alpine Linux}
\label{sec:org2038ce6}

Instalamos los siguientes paquetes por si acaso:

\begin{verbatim}
apk add grub-efi efibootmgr
\end{verbatim}

Corremos el comando:

\begin{verbatim}
BOOTLOADER=grub setup-disk -m sys /mnt
\end{verbatim}

Al finalizar, reiniciamos y elegimos la nueva entrada en nuestro sistema UEFI. El resultado final es Alpine Linux instalado en bare-metal.

El tiempo de instalación final hasta el reinicio fue: \textbf{18:57 minutos}

\section{Fedora}
\label{sec:orga68a3e5}

\subsection{Administración y ampliación de almacenamiento}

En el caso de Fedora Workstation, el sistema operativo ya se encontraba instalado previamente en el equipo con una partición de menor tamaño, por ello solo fue necesario realizar una ampliación del espacio hasta alcanzar \textbf{64 GB} disponible para cumplir con los requerimientos de almacenamiento solicitados en la práctica.
\begin{figure}[H]
    \centering
    \includegraphics[width=0.55\textwidth]{tiempo-de-ejecución/img/mnt.jpeg}
    \caption{Ampliación de espacio.}
\end{figure}
Para ello, primero se identificaron las particiones existentes dentro del disco duro utilizando la herramienta gráfica \textbf{Disks} de GNOME, la cual permite visualizar de forma más clara la distribución del almacenamiento y las particiones disponibles.

La partición correspondiente al sistema Fedora era \textbf{/dev/sda6}, configurada con el sistema de archivos \textbf{Btrfs} y un tamaño inicial de \textbf{64 GB}. Posteriormente, se liberó espacio disponible dentro del disco principal para asignarlo al sistema operativo.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.55\textwidth]{tiempo-de-ejecución/img/particion1T.jpeg}
    \caption{Administración de particiones y redimensionamiento del volumen en Fedora.}
\end{figure}

En la imagen anterior puede observarse el proceso de redimensionamiento de la partición, donde se asignó espacio libre adicional al volumen utilizado por Fedora.

Posteriormente, se realizaron pruebas de montaje manual desde la terminal para verificar las particiones disponibles y comprobar el acceso a los sistemas de archivos.

El comando utilizado fue:

\begin{verbatim}
sudo mount /dev/sda4 /mnt2
\end{verbatim}

Este comando intenta montar la partición \textbf{/dev/sda4} sobre el directorio \textbf{/mnt2}. Sin embargo, el sistema devolvió un error debido a que dicha partición no correspondía a un sistema de archivos compatible o válido para el montaje esperado.

Después de identificar correctamente la partición del sistema Fedora, se utilizó:

\begin{verbatim}
sudo mount /dev/sda6 /mnt2
\end{verbatim}

Con este comando se montó correctamente la partición principal de Fedora en el directorio temporal \textbf{/mnt2}, permitiendo acceder a los archivos internos del sistema.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.70\textwidth]{tiempo-de-ejecución/img/terminal.jpeg}
    \caption{Montaje manual de particiones y verificación del sistema de archivos en Fedora.}
\end{figure}

Una vez montada la partición, se utilizó el comando:

\begin{verbatim}
ls -la
\end{verbatim}

para listar los archivos y directorios ocultos presentes dentro del sistema montado. Esto permitió verificar la existencia de directorios importantes como \textbf{home00} y \textbf{root00}.

Posteriormente, se ejecutó:

\begin{verbatim}
cat root00/etc/passwd
\end{verbatim}

con el objetivo de visualizar el contenido del archivo \textbf{/etc/passwd}, el cual almacena la información básica de los usuarios registrados dentro del sistema Linux. Esto permitió comprobar correctamente la estructura interna del sistema Fedora montado.


El espacio utilizado por la instalación base de Fedora Workstation fue aproximadamente de \textbf{18 GB}, considerando el entorno gráfico y las aplicaciones preinstaladas.

El tiempo de inicio del sistema fue de aproximadamente \textbf{27 segundos}, medidos desde el encendido del equipo hasta la aparición de la pantalla de inicio de sesión gráfica.
