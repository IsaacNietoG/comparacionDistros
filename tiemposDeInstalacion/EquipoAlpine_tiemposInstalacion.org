* Alpine Linux

** Montaje del sistema raíz

El mecanismo de instalación de Alpine Linux está constituido por dos scripts alternativa:

- setup-alpine, el cual puede ser aplicado sobre un disco duro entero y realizará la instalación automáticamente, según las especificaciones dadas por el usuario a través de variables de entorno y preguntas interactivas del mismo script.

- setup-disk, el cual es aplicado sobre un arbol de archivos ya montado en */mnt*. Sobre este sistema se detectan las particiones utilizadas y se realiza la instalación sobre estas mismas.

Para nuestro caso de uso, el segundo script será el usado. Por lo que lo primero q que debemos de hacer es crear el arbol raíz en /mnt.

Montamos nuestra partición raíz sobre /mnt con:

#+begin_src bash 
mount -t ext4 <particion> /mnt
#+end_src

Luego, dentro de este, creamos la carpeta /boot con:

#+begin_src bash
mkdir -p /mnt/boot/efi
mount -t vfat <particion> /mnt/boot/efi
#+end_src

Y finalmente, cargamos la partición swap con:

#+begin_src bash 
swapon <particion>
#+end_src

Una vez hecho esto, tenemos preparado el arbol de archivos sobre el que se instalará el sistema operativo.

** Instalación de Alpine Linux

Instalamos los siguientes paquetes por si acaso:

#+begin_src bash
apk add grub-efi efibootmgr
#+end_src

Corremos el comando:

#+begin_src bash
BOOTLOADER=grub setup-disk -m sys /mnt
#+end_src

Al finalizar, reiniciamos y elegimos la nueva entrada en nuestro sistema UEFI. El resultado final es Alpine Linux instalado en bare-metal.

El tiempo de instalación final hasta el reinicio fue: *18:57 minutos*

** Sistema instalado

La versión de Alpine Linux instalada fue la más básica posible, con nada más que los paquetes por default, sin entorno gráfico ni ningún adicional.

El tiempo de arranque de esta instalación es de *26.79 segundos*, medidos desde el primer fotograma de la pantalla inicial de la UEFI. Si medimos el tiempo considerando solamente desde que el hardware le pasa el control a la partición de arranque del sistema, esto nos da *21.12 segundos*

** Administración y ampliación de almacenamiento

En el caso de Fedora Workstation, el sistema operativo ya se encontraba instalado previamente en el equipo con una partición de menor tamaño. Por ello, únicamente fue necesario realizar una ampliación del espacio hasta alcanzar *64 GB* disponibles para cumplir con los requerimientos de almacenamiento solicitados en la práctica.

#+CAPTION: Ampliación de espacio.
[[file:tiempo-de-ejecución/img/mnt.jpeg]]

Para ello, primero se identificaron las particiones existentes dentro del disco duro utilizando la herramienta gráfica *Disks* de GNOME, la cual permite visualizar de forma más clara la distribución del almacenamiento y las particiones disponibles.

La partición correspondiente al sistema Fedora era */dev/sda6*, configurada con el sistema de archivos *Btrfs* y un tamaño inicial de *64 GB*. Posteriormente, se liberó espacio disponible dentro del disco principal para asignarlo al sistema operativo.

#+CAPTION: Administración de particiones y redimensionamiento del volumen en Fedora.
[[file:tiempo-de-ejecución/img/particion1T.jpeg]]

En la imagen anterior puede observarse el proceso de redimensionamiento de la partición, donde se asignó espacio libre adicional al volumen utilizado por Fedora.

Posteriormente, se realizaron pruebas de montaje manual desde la terminal para verificar las particiones disponibles y comprobar el acceso a los sistemas de archivos.

El comando utilizado fue:

#+BEGIN_SRC bash
sudo mount /dev/sda4 /mnt2
#+END_SRC

Este comando intenta montar la partición */dev/sda4* sobre el directorio */mnt2*. Sin embargo, el sistema devolvió un error debido a que dicha partición no correspondía a un sistema de archivos compatible o válido para el montaje esperado.

Después de identificar correctamente la partición del sistema Fedora, se utilizó:

#+BEGIN_SRC bash
sudo mount /dev/sda6 /mnt2
#+END_SRC

Con este comando se montó correctamente la partición principal de Fedora en el directorio temporal */mnt2*, permitiendo acceder a los archivos internos del sistema.

#+CAPTION: Montaje manual de particiones y verificación del sistema de archivos en Fedora.
[[file:tiempo-de-ejecución/img/terminal.jpeg]]

Una vez montada la partición, se utilizó el comando:

#+BEGIN_SRC bash
ls -la
#+END_SRC

para listar los archivos y directorios ocultos presentes dentro del sistema montado. Esto permitió verificar la existencia de directorios importantes como *home00* y *root00*.

Posteriormente, se ejecutó:

#+BEGIN_SRC bash
cat root00/etc/passwd
#+END_SRC

con el objetivo de visualizar el contenido del archivo */etc/passwd*, el cual almacena la información básica de los usuarios registrados dentro del sistema Linux. Esto permitió comprobar correctamente la estructura interna del sistema Fedora montado.

El espacio utilizado por la instalación base de Fedora Workstation fue aproximadamente de *18 GB*, considerando el entorno gráfico y las aplicaciones preinstaladas.

El tiempo de inicio del sistema fue de aproximadamente *27 segundos*, medidos desde el encendido del equipo hasta la aparición de la pantalla de inicio de sesión gráfica.
